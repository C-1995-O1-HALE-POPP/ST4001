\section*{A1 Dataset Pipeline}
\addcontentsline{toc}{section}{A1 Dataset Pipeline}

The dataset pipeline starts from raw MIMIC-IV notes and structured tables, converts each admission into a typed clinical event stream, and then derives the current-visit interaction context used by the agentic decision task. The patient-level sequence file \texttt{bench\_data/patients\_sequence/P000001\_sequenced.json} contains 813 ordered events for Patient~P000001, including demographics, admissions, labs, microbiology, imaging, medications, procedures, and discharges.

\subsection*{A1.1 Raw Discharge Note Example}
\addcontentsline{toc}{subsection}{A1.1 Raw Discharge Note Example}

The raw note table \texttt{raw\_data/note/discharge.csv} stores one discharge summary per row. A representative row has the schema \texttt{note\_id}, \texttt{subject\_id}, \texttt{hadm\_id}, \texttt{note\_type}, \texttt{note\_seq}, \texttt{charttime}, \texttt{storetime}, and \texttt{text}.

\begin{promptblock}{Raw Discharge Note Row}
note_id: 10000032-DS-21
subject_id: 10000032
hadm_id: 22595853
note_type: DS
charttime: 2180-05-07 00:00:00
storetime: 2180-05-09 15:26:00
text: Name: ___  Unit No: ___
Admission Date: ___  Discharge Date: ___
Service: MEDICINE
Allergies: No Known Allergies / Adverse Drug Reactions
Chief Complaint: Worsening ABD distension and pain
Major Surgical or Invasive Procedure: Paracentesis
History of Present Illness: ___ HCV cirrhosis c/b ascites,
hiv on ART, h/o IVDU, COPD, bipolar, PTSD, presented from OSH ED
with worsening abdominal distension and pain ...
\end{promptblock}

During preprocessing, admission-facing fields such as chief complaint and history of present illness initialize the current visit, while discharge-facing fields become visit-level Note Memory for later visits.

\section*{A2 Memory Construction}
\addcontentsline{toc}{section}{A2 Memory Construction}

LongMedBench uses three memory forms. Note Memory stores previous-visit discharge summaries, Event Memory stores previous-visit structured event streams, and Context Memory stores the dialogue-style state of the current visit before a decision point.

\subsection*{A2.1 LLM Convert}
\addcontentsline{toc}{subsection}{A2.1 LLM Convert}

Context Memory is the only memory type that uses an LLM during construction. The code first maps structured events to semantic actions, then asks the LLM to write the assistant-side rationale field under strict grounding constraints.

\begin{promptblock}{ContextMemory LLM Conversion Prompt}
You write the 'reason' field for the next assistant action in a
doctor-agent dialogue.

Hard constraints:
- You may ONLY use information present in the provided prefix_messages.
- Do NOT invent new patient facts, findings, diagnoses, or outcomes.
- Do NOT mention results of the current action (results are not available yet).
- Keep it 1-2 sentences, not exceeding 50 words, workflow-rationale focused.

Output ONLY JSON: {"reason": string, "constraints_ok": boolean}.

User payload:
{
  "prefix_messages": <system, admission, and previous interaction messages>,
  "current_decision": {"action": <action>, "args": <arguments>}
}
\end{promptblock}

The remaining memory types are constructed automatically. Note Memory is produced by extracting the complete discharge-note text from previous visits. Event Memory is produced by chronological serialization of structured EHR events. Neither requires LLM rewriting.

For T3-A questions, candidate answers are collected from future events in a 24-hour window after the current timestamp and weighted by temporal decay. T3-D instead compares the current timestamp with the discharge time using a six-hour horizon.

\section*{A4 Agent Configuration}
\addcontentsline{toc}{section}{A4 Agent Configuration}

Each evaluation condition is defined by two axes: the historical memory source, either Note Memory or Event Memory, and the delivery strategy, either long-context prompting, retrieval-augmented generation (RAG), or Mem0-style managed memory. The current-visit Context Memory is always included as the immediate interaction state.

\section*{A5 Case Study}
\addcontentsline{toc}{section}{A5 Case Study}

This case study shows how one decision instance is answered under different memory configurations. It includes the original Context Memory excerpt and the original question, but omits file indexes, retrieval indexes, and source-path metadata.

\paragraph{Case identifier and task.}
The instance is \texttt{P000001-V12-T0002-T3-N-0}, a T3-N Next Action Prediction question at \texttt{2121-12-16 19:10:00}. The ground truth is \texttt{order\_imaging} with weight 1.0.

\begin{promptblock}{Original ContextMemory Excerpt}
system: You are a doctor agent. You can perform these semantic actions:
ask_question, order_labs, order_microbiology, order_imaging,
perform_procedure, medication, discharge. The environment returns only
information that exists in this admission record; otherwise it returns
not_available. Do not invent new findings.

user: Gender: M. Age at first visit: 48. Language: Spanish. Marital
status: SINGLE. Race: HISPANIC/LATINO - DOMINICAN. Chief complaint:
Chest Pa. HPI: ___ year old male with HCV and EtOH cirrhosis on the
transplant list, complicated by ___ s/p TACE x2 and RFA and ascites,
and recurrent C. difficile colitis, presents with increasing abdominal
distention, bilateral lower extremity swelling, and occasional
non-pleuritic, non-exertional chest pain for 3 days. ...

assistant: {
  "reason": "The patient has atypical chest pain and bilateral lower
             extremity edema, warranting a chest X-ray to evaluate for
             possible pulmonary causes.",
  "action": "order_imaging",
  "args": {"modality": "xray", "target": "chest",
           "timestamp": "2121-12-16 19:05:00"}
}

environment: chest xray -> Limited, negative.

assistant: {
  "reason": "The patient has bilateral lower extremity edema,
             warranting a Doppler ultrasound to evaluate for deep vein
             thrombosis.",
  "action": "order_imaging",
  "args": {"modality": "doppler", "target": "free text body part",
           "timestamp": "2121-12-16 19:10:00"}
}

environment: free text body part doppler -> Addendum: A limited Color
Doppler and Spectral Analysis Study was also performed.
\end{promptblock}

\begin{promptblock}{Original Question}
qid: P000001-V12-T0002-T3-N-0
visit_id: P000001-V12
timestamp: 2121-12-16 19:10:00
qtype: T3-N
question: Which of the next step is most appropriate?
options:
  - perform_procedure
  - medication
  - ask_question
  - order_microbiology
  - order_labs
  - order_imaging
answer: {"order_imaging": 1.0}
\end{promptblock}

The case highlights two behaviors. First, RAG succeeds here because retrieved evidence reinforces the local imaging trajectory. Second, Mem0 can retrieve clinically plausible but temporally mismatched memories; those memories produce coherent rationales but answer a different clinical situation from the current decision point.
