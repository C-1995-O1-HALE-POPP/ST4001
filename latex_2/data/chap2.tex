\chapter{Methodology \& Experiments}

This chapter presents the LongMedBench long-horizon reasoning pipeline: from raw EHR data to automatically constructed decision queries for interactive agent evaluation. Section~\ref{sec:pipeline} describes the data processing pipeline. Section~\ref{sec:memory} presents the three-tier memory architecture. Section~\ref{sec:task} formulates the Long-Horizon Reasoning Task as a template-driven clinical reasoning problem. Section~\ref{sec:expsetup} details the experimental design. LongMedBench is constructed from MIMIC-IV v3.1 \cite{PhysioNet-mimiciv-3.1}, a publicly available de-identified database containing comprehensive clinical data for over 100,000 patients at Beth Israel Deaconess Medical Center (2008--2022). To convert this database into an interactive agent-compatible environment, we design a three-stage pipeline (Figure~\ref{fig:pipeline}).


\subsection{Data Preprocessing \& Event Filtering}

The preprocessing stage converts EHR records of MIMIC-IV \cite{PhysioNet-mimiciv-3.1} into the longitudinal substrate used by the memory architecture. We retain clinically salient structured events from the \texttt{hosp} module, including abnormal laboratory results, microbiology, medications, diagnoses, procedures, and pharmacy records. Each hospitalization is anchored by an admission event and a discharge event, with matched discharge summaries from the \texttt{note} module. Meanwhile, radiology reports from the \texttt{note} module are parsed as imaging events for downstream memory construction. This filtering keeps the data representation compact enough for controlled evaluation while preserving the information needed to compare memory granularity.

\paragraph{Resulting Cohort.} The filtered dataset comprises 355 patients with 6,999 inpatient visits. Mean visits per patient: 19.72 (median = 18.00, SD = 5.72). Mean medical events per visit: 44.91 (median = 25.00, SD = 70.32). The distribution of data is shown in Figure~\ref{fig:visit_distribution}. This dense multi-session structure provides the longitudinal basis for evaluating memory retention and temporal reasoning.

\subsection{Event Stream Construction}

We transform filtered records into a unified, time-ordered event stream for each patient. A medical event is any discrete clinical occurrence extracted from the structured hospitalization data. Formally, the $k_\text{th}$ event in $\mathcal{V}_i$ is $E^k_i := \{a^k_i, t^k_i, p^k_i, o^k_i\}$, where $a^k_i$ is the action type from the clinical action space $\mathcal{A}$, \textit{including imaging, lab tests, medication, etc.}, $t^k_i$ is the timestamp, $p^k_i$ is the action parameter \textit{(e.g., CT, X-ray, etc. for imaging)}, and $o^k_i$ is the observation result \textit{(e.g., a radiology report)}.

We define a visit as the period spanning from an admission event $E^{adm}_i$ to the corresponding discharge event $E^{dis}_i$. All other structured events occurring between these two timestamps are collected and ordered chronologically. Formally, each visit $\mathcal{V}_i$ can be represented as $\mathcal{V}_i = \{E^{adm}_i, E^1_i, E^2_i, \dots, E^m_i, E^{dis}_i\}$. The complete patient stream can therefore be denoted as $\mathcal{S_P} = \{\mathcal{V}_1, \mathcal{V}_2, \dots, \mathcal{V}_n\}$, where $n$ is the total number of visits.

\section{Three-Tier Memory Architecture}
\label{sec:memory}

To provide the agents with controlled provision of historical information, we define a three-tier memory architecture that systematically varies the granularity of historical information. Given the earliest historical visit $\mathcal{V}_i$, the current visit $\mathcal{V}_j$, and the current timestamp $T$, the agent's memory access is strictly bounded to prevent event leakage.

\subsection{Note Memory}
\label{sec:notemem}

$\mathcal{M}^{(i,j)}_N := \{N_i, N_{i+1}, \dots, N_{j-1}\}$, which contains continuous visit-level discharge summaries from $\mathcal{V}_i$ to $\mathcal{V}_{j-1}$. Note Memory is compact and preserves clinical narrative structure, emphasizing the global trajectory, but discards fine-grained temporal and event-level details.

\subsection{Event Memory}
\label{sec:eventmem}

$\mathcal{M}^{(i,j)}_E := \bigcup_{k=i}^{j-1} \mathcal{V}_k$, which contains all clinical events from $\mathcal{V}_i$ to $\mathcal{V}_{j-1}$. Events are flattened into a single chronological sequence, discarding visit-level boundaries and forming a unified trajectory. Event Memory is information-complete, and its volume creates a natural stress test for both long-context capabilities and retrieval strategies.

\subsection{Contextual Memory}
\label{sec:contextmem}

$\mathcal{M}_C^{(j,T)} := \{(r_0, m_0), (r_1, m_1), \dots, (r_P, m_P)\}$, where $r_k$ and $m_k$ denote the dialog role (\texttt{assistant} or \texttt{user}) and message content, respectively. Through an LLM-based rewriting template, any event $E_j^k$ in $\mathcal{V}_j$ with timestamp $t_j^k < T$ is transformed into a conversation trajectory between the agent and the clinical environment, simulating the agent's immediate interaction state up to the decision point.

Contextual Memory represents the baseline information always available to the agent at the current interaction state. Historical memory ($\mathcal{M}^{(i,j)}_N$ or $\mathcal{M}^{(i,j)}_E$) is the experimental variable, enabling isolation of the marginal contribution of history beyond the immediate clinical context.

\section{Long-Horizon Reasoning Task}
\label{sec:task}

\begin{figure}
    \centering
    \includegraphics[width=\textwidth]{images/decision_making.pdf}
    \caption{Long-Horizon Reasoning Task interface for memory-augmented clinical agents.}
    \label{fig:decision_task}
\end{figure}

\subsection{Task Formulation}

For a sampled ground-truth event $E_i^{GT}$ in the current visit $\mathcal{V}_i$ with timestamp $t_i^{GT}$, the working context is defined as $S=\mathcal{M}_C^{(i,t_i^{GT})}$, containing all interaction history available before the target event. To ensure sufficient historical trajectory length, we only sample rounds where $n \ge 15$. The long-horizon memory is denoted as $M=\mathcal{M}^{(1,n)}_E$ or $M=\mathcal{M}^{(1,n)}_N$, depending on whether Event Memory or Note Memory is injected. The multiple-choice query $Q$ is automatically constructed from templates based on the type of the question and the ground-truth event, which makes the task scalable while preserving a fixed decision interface across patients and visits. The three subtasks are defined as follows:
\begin{itemize}
    \item   \textbf{Next Action Prediction} asks the agent to predict the future event action type $a_i^{GT}$ from the clinical action space $\mathcal{A}$.
    \item   \textbf{Argument Prediction} asks the agent to predict the target event argument $p_i^{GT}$, such as which laboratory panels or imaging modality are most appropriate.
    \item   \textbf{Discharge Decision} asks whether the patient should be discharged within $6$ hours, yielding a binary Yes/No decision.
\end{itemize}

At inference time, the agent receives $S$, an instructional prompt, $M$, and $Q$. The task is formulated as a Long-Horizon Reasoning Task because the model is explicitly required to reason before committing to the next clinical decision. Given the current context and injected historical memory, the model first produces an explicit chain-of-thought-style rationale in the JSON \texttt{reason} field and then returns the selected decision in the JSON \texttt{answer} field. This reason-then-act interface follows the general design philosophy of ReAct-style agents, where reasoning is used to support action selection \cite{yao2022react}. The design also allows for qualitative analysis of the reasoning process, enabling identification of specific failure modes such as retrieval errors, temporal misjudgments, or flawed inference as shown in Appendix~A.5.

\subsection{Evaluation Metrics}

Next Action Prediction and Discharge Decision are evaluated by classification accuracy, while Argument Prediction is evaluated by top-$k$ accuracy because multiple arguments may be clinically appropriate. Discharge Decision is evaluated directly as binary classification accuracy. For Next Action Prediction and Argument Prediction, we additionally apply a time-decay scoring mechanism to recognize decisions matching action occurs within 24 hours after the sampled timestamp. This design evaluates whether the model can make a temporally appropriate decision, rather than whether it can reproduce one exact event index. Let
\[
\mathcal{E}_{24h}=\{E_i^k \mid t_i^k\in[t_i^{GT},t_i^{GT}+24\mathrm{h}]\}.
\]
The score is
\[
S=\max_{E_i^k\in\mathcal{E}_{24h}}
\exp\{-\lambda(t_i^k-t_i^{GT})\}\cdot\operatorname{Score}(y_i^k,\hat{y}),
\quad
\lambda=\frac{\ln(100)}{24},
\]
where the decay factor decreases from $1.0$ at the target timestamp to $0.01$ after $24$ hours, $\operatorname{Score}(\cdot)$ is classification accuracy for Next Action Prediction and top-$k$ accuracy for Argument Prediction. 

\section{Experimental Setup}
\label{sec:expsetup}

\subsection{Models}

We evaluate three state-of-the-art LLMs: GPT-5-mini \cite{gpt52}, DeepSeek-v3.2 \cite{deepseekai2025deepseekv32}, and Qwen-Turbo \cite{qwen3}. All are evaluated through their respective APIs with standardized prompts and default model configurations.

\subsection{Agent Architectures}

Given the fixed memory modules $\{\mathcal{M}^{(i,j)}_N, \mathcal{M}^{(i,j)}_E, \mathcal{M}_C^{(j,T)}\}$, three strategies control how historical memory is provided to the agent: \textbf{(1) Long-Context LLM.} $\mathcal{M}^{(i,j)}_N$ or $\mathcal{M}^{(i,j)}_E$ is provided in full within the model's context window alongside $\mathcal{M}_C^{(j,T)}$, testing native long-context attention without external retrieval assistance. \textbf{(2) RAG.} $\mathcal{M}^{(i,j)}_N$ or $\mathcal{M}^{(i,j)}_E$ is externally indexed. At decision time, $\mathcal{M}_C^{(j,T)}$ serves as the retrieval query; top-$k$ relevant chunks are retrieved and concatenated with the context. \textbf{(3) Agent Memory System.} We integrate the Mem0 framework \cite{mem0_reference}, providing a persistent, evolvable memory store to the agent. The agent can add, search, and update memory across a patient's trajectory, actively managing what is stored and retrieved. The design tests whether agentic memory system outperforms passive retrieval for action-oriented reasoning.

\subsection{Experimental Conditions}

We design a factorial ablation varying four factors:

\begin{enumerate}
    \item \textbf{Memory Granularity:} Note Memory vs.\ Event Memory.
    \item \textbf{Provision Strategy:} Naive Long-Context vs.\ RAG vs.\ Mem0.
    \item \textbf{Historical Depth ($n$):} Number of injected historical visits: $n \in \{0, 2, 5\}$. $n=0$ provides Context Memory $\mathcal{M}_C^{(j,T)}$ only with no history as baseline.
    \item \textbf{Reasoning Mode:} Standard vs.\ model-native thinking mode (DeepSeek-v3.2 only), where the thinking condition uses DeepSeek's native deep exploration/reasoning mode rather than an ordinary prompt-level chain-of-thought exemplar.
\end{enumerate}

Qwen-Turbo serves as the primary ablation vehicle; key conditions are replicated on GPT-5-mini and DeepSeek-v3.2 for cross-model validation.

\subsection{Model Level Patterns}

Table~\ref{tab:decision_overall} reports the long-context LLM performance, where each model is allowed to use the available patient history within a maximum 128K-token context. The strongest overall result is obtained by GPT-5-mini with Event Memory (Avg = 0.452), followed by GPT-5-mini with Note Memory (Avg = 0.445) and DeepSeek-v3.2 with Event Memory (Avg = 0.439). This ranking supports the LongMedBench observation that structured event memory can outperform note memory for several models in decision-making, but the advantage is not universal: Qwen-Turbo performs better with Note Memory, and DeepSeek-v3.2-thinking remains nearly unchanged across memory types.

At the subtask level, Discharge Decision reaches approximately 0.58--0.63 because it is a binary discharge decision, whereas Next Action Prediction remains around 0.28--0.34 because selecting the next action type requires finer-grained clinical planning. Argument Prediction lies in the middle, reflecting the difficulty of selecting appropriate event arguments such as laboratory panels or imaging targets. 

Event Memory slightly improves Argument Prediction in most models, suggesting that fine-grained historical events help parameter selection, but the gains do not consistently translate into higher average decision-making performance. DeepSeek-v3.2's thinking mode does not improve decision-making performance in this benchmark. Under Event Memory, the average score drops from 0.439 to 0.419, and Discharge Decision decreases from 0.607 to 0.593. This suggests that extended model-native reasoning may make the model more conservative or distract it from committing to the multiple-choice decision, especially when the task requires selecting an actionable next step rather than explaining uncertainty.

\subsection{Memory Ablations: Visit Injection and Retrieval Architectures}



Table~\ref{tab:decision_ablation} reports the Qwen-Turbo ablation used to test historical depth and memory-augmentation strategies. Increasing the number of injected visits does not produce monotonic gains, and retrieval-based architectures remain close to or below the naive baseline.


\textbf{Finding.} The ablation results revise the earlier interpretation: the negative result is not specific to Discharge Decision, but spans the broader decision-making suite. Even when historical information is made available through longer visit injection, RAG, or Mem0, models do not systematically improve on Argument Prediction, Discharge Decision, or Next Action Prediction. The $n=0$ baseline remains competitive, indicating that agents rely heavily on immediate context and struggle to convert retrieved history into better decisions.
